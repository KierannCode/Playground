Detailed Description

Thread-safe polymorph queue.

A thread-safe implementation of a polymorph queue with staged popping :
This is a FIFO structure you can push elements into and pop elements off.
Consecutive pops empty the queue, starting with the first pushed elements.
The last element popped can be restored at the beginning of the queue.
This strucutre is thread-safe, meaning that most of its methods can be called simultaneously in any thread.

◆ pop()

OSDL::Data* OSDL::AtomicQueue::pop ( )

Pops data at the beginning of the queue.

Removes and returns the data at the beginning of the queue.
The last popped element is temporarily saved and can be pushed back at the beginning of the queue using 'unPop'.
Only the last popped element is saved, calling this method multiple times effectively empties the queue. This method is thread-safe and can be called safely in any thread.

void OSDL::AtomicQueue::unPop ( )

Restores the last popped element.

Restores the last popped element at the beginning of the queue.
This method is not thread-safe and must be called in the same thread that called 'pop' for the last time. Calling this method when no element was popped will throw an exception.
