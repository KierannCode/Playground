Detailed Description

Implementation of a mutex using SDL primitives.

This implementation is the Object Oriented version of SDL_mutex. A mutex shall be used to prevent concurrent access to shared ressources between several threads. The mutex must be locked right before using the ressources, and unlocked right after. This implementation supports recursive locking.

◆ lock()

void OSDL::Mutex::lock ( )

Locks the mutex.

When a mutex is locked in a thread, this method will pause any other thread trying to lock it, until it is unlocked in the thread where it has been locked. If a mutex is locked several times in a row in the same thread, it must be unlocked the same amount of times to be available in other threads, this is called recursive locking.

void OSDL::Mutex::unlock ( )

Unlocks the mutex.

This method must be called when the mutex is locked in order to unlock it. If the mutex was locked several times in a row in the same thread, it must be unlocked the same number of times to be available in other threads. Calling this method when the mutex is not locked results in undefined behavior.
